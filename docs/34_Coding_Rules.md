# Reglas de Desarrollo

Nunca usar any.

Nunca escribir números mágicos.

Nunca hardcodear nombres.

Nunca hardcodear probabilidades.

Nunca mezclar lógica y UI.

Nunca acceder directamente a la base de datos desde React.

Nunca modificar el estado directamente.

Todo mediante Store.

Todo el Game Engine debe funcionar sin React.

Todo el contenido debe venir de JSON.

Todos los SVG deben ser reutilizables.

No crear componentes de más de 300 líneas.

No crear funciones de más de 80 líneas.

Todo debe ser TypeScript estricto.

Todo debe tener tipos.

Todo debe tener comentarios únicamente cuando aporten valor.

Todo debe ser modular.

Todo debe ser reutilizable.

Todo debe poder testearse.

Todo debe ser escalable.

Todo preparado para añadir nuevas regiones.
