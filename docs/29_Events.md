# Sistema de Eventos

## Filosofía

Los eventos convierten cada Run en una aventura.

No deben sentirse repetitivos.

No existen respuestas correctas.

Cada decisión tiene ventajas y desventajas.

El jugador nunca conoce exactamente el resultado.

---

## Frecuencia

Cada Run debe contener entre:

8 y 15 eventos.

Nunca menos.

Nunca demasiados.

## Ejemplo: Anciano Perdido

Un anciano te pide ayuda para cruzar el bosque.

Opciones

Resultados

Nada
